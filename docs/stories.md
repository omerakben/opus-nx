# Prioritized User Stories

## P0 - Setup and Access

1. As a new contributor, I can run `pnpm setup` to bootstrap local env files with safe defaults.
2. As a new contributor, I can run `pnpm setup:verify` to confirm required credentials are valid.
3. As a user, I can access `/` without provider credentials and see setup guidance.
4. As an authenticated user, I can access `/workspace` for reasoning workflows.

## P0 - Research Core

1. As a researcher, I can persist reasoning graphs and inspect decision points.
2. As a researcher, I can rerun hypotheses and compare outcomes in lifecycle workflows.
3. As a researcher, I can retrieve similar artifacts across sessions.

## P1 - Contributor Experience

1. As a contributor, I can follow one canonical docs map and avoid stale historical guidance.
2. As a contributor, I can rely on explicit PR/test expectations from CONTRIBUTING docs.
3. As a contributor, I can understand expected behavior standards from CODE_OF_CONDUCT.

## P1 - Product UX

1. As a visitor, I can understand current platform scope from the public landing page.
2. As a visitor, I can identify the primary CTA to run locally with my own credentials.
3. As a visitor, I can see demo mode is optional and disabled by default.

## P2 - Future Scope

1. As a team admin, I can manage per-user credentials in a multi-tenant hosted mode.
2. As an organization, I can enforce tenant isolation and key rotation policy.
